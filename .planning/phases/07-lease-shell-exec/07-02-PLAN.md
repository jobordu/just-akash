<objective>
Extend LeaseShellTransport.exec() to silently re-authenticate and reconnect when the provider closes the WebSocket due to JWT expiry, then continue streaming output until the command completes. Add unit tests proving the reconnect path.

Purpose: Fulfils LSHL-03 (token expiry during long sessions triggers automatic re-authentication without dropping the user session). Long-running commands (builds, downloads) must survive the 30-second default JWT TTL even if the server ignores the `ttl=3600` request.
Output: exec() that retries on auth-close (WebSocket close codes 4001/4003, or `ConnectionClosedError` containing "expired"/"unauthorized") by fetching a fresh JWT and reopening the connection. Tests confirm the reconnect fires exactly once on expiry and the final exit code is propagated correctly.

Extend `just_akash/transport/lease_shell.py` to handle JWT expiry during exec. The strategy is:

1. Add a module-level constant `MAX_RECONNECT_ATTEMPTS = 3` (guards against infinite reconnect loops).
2. Extract the inner WebSocket receive loop into a private method `_exec_with_refresh(url_template: str, command: str) -> int` that takes the base URL without query params. This method owns the reconnect loop.
3. `exec()` becomes a thin wrapper that calls `_exec_with_refresh()`.

**Reconnect detection:** Treat the connection as auth-expired when `ConnectionClosedError` is raised AND the close `rcvd.code` is `4001` or `4003`, OR the close reason string contains `"expired"` or `"unauthorized"` (case-insensitive). Also treat `RuntimeError` from frame code 103 if its message contains `"expired"` or `"unauthorized"`.

**`_exec_with_refresh()` pseudocode:**

```python
MAX_RECONNECT_ATTEMPTS = 3

def _exec_with_refresh(self, command: str) -> int:
    """Execute command with automatic JWT refresh on token expiry.

    On ConnectionClosedError with auth close code (4001/4003) or 'expired'/'unauthorized'
    in reason, fetches a new JWT and reopens the connection. Repeats up to
    MAX_RECONNECT_ATTEMPTS times. Accumulated stdout/stderr already written
    to sys.stdout.buffer/sys.stderr.buffer — nothing is replayed.
    """
    attempts = 0
    exit_code = 0

    while attempts < MAX_RECONNECT_ATTEMPTS:
        jwt = self._fetch_jwt()
        params = urllib.parse.urlencode({
            "cmd": command,
            "service": self._service,
            "tty": "false",
            "stdin": "false",
        })
        url = f"{self._ws_url}?{params}"
        headers = {"Authorization": f"Bearer {jwt}"}
        ssl_ctx = self._make_ssl_context()

        try:
            with connect(
                url,
                additional_headers=headers,
                ssl=ssl_ctx,
                compression=None,
                open_timeout=30,
                ping_interval=20,
                ping_timeout=20,
            ) as ws:
                while True:
                    try:
                        frame = ws.recv(timeout=300)
                    except ConnectionClosedOK:
                        return exit_code
                    except ConnectionClosedError as exc:
                        if _is_auth_expiry(exc):
                            break  # retry outer loop
                        raise  # non-auth close: propagate
                    result = self._dispatch_frame(frame)
                    if result is not None:
                        return result  # exit code received: done
        except RuntimeError as exc:
            if _is_auth_expiry_message(str(exc)):
                pass  # fall through to retry
            else:
                raise
        attempts += 1

    raise RuntimeError(
        f"Failed to re-authenticate after {MAX_RECONNECT_ATTEMPTS} attempts. "
        "Check that AKASH_API_KEY is valid and the deployment is active."
    )
```

Add two module-level helper functions (not methods — simpler to test in isolation):

```python
def _is_auth_expiry(exc: ConnectionClosedError) -> bool:
    """Return True if close event indicates JWT expiry or auth failure."""
    rcvd = getattr(exc, "rcvd", None)
    if rcvd is not None:
        code = getattr(rcvd, "code", None)
        if code in (4001, 4003):
            return True
        reason = getattr(rcvd, "reason", "") or ""
        if _is_auth_expiry_message(reason):
            return True
    # Fallback: check exception string representation
    return _is_auth_expiry_message(str(exc))


def _is_auth_expiry_message(msg: str) -> bool:
    """Return True if message text indicates auth expiry."""
    lower = msg.lower()
    return "expired" in lower or "unauthorized" in lower or "token" in lower
```

Update `exec()` to delegate to `_exec_with_refresh()`:

```python
def exec(self, command: str) -> int:
    if self._ws_url is None or self._service is None:
        self.prepare()
    return self._exec_with_refresh(command)
```

**Important:** Do NOT replay stdout/stderr on reconnect. Output already written to `sys.stdout.buffer`/`sys.stderr.buffer` is delivered — reconnect only resumes receiving new frames. This is the correct behavior for long-running streaming commands.
