2. **Add `_make_ssh_transport` helper** near the top of the dispatch section (before `if args.command == "deploy":`):

```python
def _make_ssh_transport(client, dseq_arg, key_arg):
    """Resolve deployment and return a prepared SSHTransport."""
    from .transport import make_transport

    dseq = _resolve_deployment(client, dseq_arg)
    deployment = client.get_deployment(dseq)
    transport = make_transport(
        "ssh",
        dseq=dseq,
        api_key=client.api_key,
        deployment=deployment,
        ssh_key_path=key_arg or "",
    )
    transport.prepare()
    return dseq, transport
```

3. **Update `exec` handler** to route through transport when `--transport` flag is used. Keep the original SSH path as fallback to avoid breaking existing tests. The safest approach: if `args.transport == "ssh"`, use `_make_ssh_transport`; else fall through to original `_require_ssh` (which will error until lease-shell is implemented):

```python
elif args.command == "exec":
    from .api import AkashConsoleAPI

    try:
        client = AkashConsoleAPI(_require_api_key())
        if args.transport == "lease-shell":
            from .transport import make_transport
            dseq = _resolve_deployment(client, args.dseq)
            deployment = client.get_deployment(dseq)
            transport = make_transport(
                "lease-shell",
                dseq=dseq,
                api_key=client.api_key,
                deployment=deployment,
            )
            transport.prepare()  # raises NotImplementedError in Phase 6
            rc = transport.exec(args.remote_cmd)
            sys.exit(rc)
        else:
            # Default: SSH (v1.4 behavior)
            dseq = _resolve_deployment(client, args.dseq)
            ssh, ssh_cmd = _require_ssh(client, dseq, args.key)
            ssh_cmd.append(args.remote_cmd)
            print(f"Executing on {ssh['host']}:{ssh['port']}...")
            result = subprocess.run(ssh_cmd, text=True)
            sys.exit(result.returncode)
    except RuntimeError as e:
        print(f"Error: {e}", file=sys.stderr)
        sys.exit(1)
```

  <name>Task 2: Inspect console.akash.network WebSocket traffic and write PROTOCOL.md</name>
  <files>docs/PROTOCOL.md</files>
  <action>
Perform browser traffic inspection of the Akash Console to discover the lease-shell WebSocket protocol. This requires a running Akash deployment accessible via console.akash.network.

**Steps:**

1. **Open Chrome/Firefox DevTools** → Network tab → filter by "WS" (WebSocket connections)

2. **Navigate to** https://console.akash.network and open a deployment's shell tab (or use the "Shell" button on a running deployment)

3. **Observe the WebSocket connection:**
   - Note the WebSocket upgrade URL (e.g., `wss://console-api.akash.network/...`)
   - Inspect the Request Headers in the connection: look for `x-api-key`, `Authorization`, or other auth headers
   - Click the "Messages" tab to see the frame exchange

4. **Document the frames:**
   - What is sent first from the client (connection initiation)?
   - What does the server send back?
   - What does a command execution request look like?
   - What do stdout/stderr/exit frames look like?
   - How does the connection close?

5. **Write `docs/PROTOCOL.md`** capturing:
   - Exact endpoint URL pattern (with `{dseq}`, `{provider}` placeholders)
   - All HTTP headers in the WebSocket upgrade request
   - JSON message schema for each observed message type (stdin, stdout, stderr, exit, resize if present)
   - Connection lifecycle (who closes first, any ping/pong)
   - Auth token format (JWT? API key? how is it passed?)
   - Any observed error frames

6. **If a running deployment is not available:** Document what was observed from the Console UI (e.g., button locations) and note which fields could not be confirmed. Add a `## Unconfirmed Fields` section to PROTOCOL.md.

**Minimum PROTOCOL.md must contain:**
- `## Endpoint` — the WebSocket URL pattern
- `## Authentication` — how auth is conveyed
- `## Message Schema` — at least stdout/stderr/exit message types
- `## Observed at` — date of inspection
